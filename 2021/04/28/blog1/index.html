Manual for using the Authority framework

2021-04-28

Role permission control framework

Introduction

The user can configure the corresponding role name in the YML configuration file, and then add the framework’s custom annotation on the URL that needs the audit permission. The role permission control of URL resources can be realized by naming the role name with the first pathname.

How to use?

  • First, we need to add the framework into our project.

We offer three ways:

  1. Clone our open source code to your project.

You can get our framework’s source on Git-hub. The URL is https://github.com/qyd-gc/edu.nuist.authority Then paste the package frame into your project.

  1. Download the JAR package and install it in the local repository.

You can get our framework’s jar package on Git-hub. The URL is https://github.com/qyd-gc/edu.nuist.authority Then download the jar package authority-1.0.jar .You can refer to this link for how to import a JAR into your local Maven repository: https://nuist-elite.github.io/myBlog/maven/maven-jar/

  • Then, you just need to add our custom annotation @RequiredPermission before the mapping that requires audit permission and set the first field in the mapping URL to the name of the accessible role to implement the interception. This means that only users with this permission can access the URL.

  • The last thing you should notice is that you only need to add a role name field to the URL, and that field is the first field. You don’t need to worry about multiple role name fields because a role can have multiple role names, which means multiple role permissions.
